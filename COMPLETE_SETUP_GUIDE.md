## 🔧 ЧТО НУЖНО НАСТРОИТЬ

### 1. 📧 EMAIL УВЕДОМЛЕНИЯ (Resend)

#### Шаг 1: Регистрация в Resend
1. Перейдите на [resend.com](https://resend.com)
2. Зарегистрируйтесь и подтвердите email
3. Создайте API ключ

#### Шаг 2: Настройка домена
1. В Resend добавьте ваш домен (например, `energylogic.com`)
2. Настройте DNS записи (SPF, DKIM, DMARC)
3. Подтвердите домен

#### Шаг 3: Переменные окружения
```bash
# В .env.local или .env.production
RESEND_API_KEY=re_your_api_key_here
NOTIFY_SENDER_EMAIL=noreply@energylogic.com
```

#### Шаг 4: Тестирование
```bash
# Запустите тест email системы
node test_notification_system.js
```

### 2. 📱 TELEGRAM УВЕДОМЛЕНИЯ

#### Шаг 1: Создание ботов
1. **Бот для админов** (уведомления о новых заявках):
   - Создайте бота через @BotFather
   - Получите токен
   - Настройте webhook

2. **Бот для пользователей** (уведомления о ответах):
   - Создайте бота через @BotFather
   - Получите токен
   - Настройте webhook

#### Шаг 2: Переменные окружения
```bash
# Бот для админов
TELEGRAM_BOT_TOKEN=your_admin_bot_token
TELEGRAM_CHAT_ID=your_admin_chat_id

# Бот для пользователей
USER_TELEGRAM_BOT_TOKEN=your_user_bot_token
USER_TELEGRAM_BOT_USERNAME=your_user_bot_username
```

#### Шаг 3: Настройка webhook
```bash
# Для админ бота
curl -X POST "https://api.telegram.org/bot<ADMIN_BOT_TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://yourdomain.com/api/telegram/admin-webhook"}'

# Для пользовательского бота
curl -X POST "https://api.telegram.org/bot<USER_BOT_TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://yourdomain.com/api/telegram/user-webhook"}'
```

### 3. ⏰ CRON JOB ДЛЯ ОБРАБОТКИ УВЕДОМЛЕНИЙ

#### Вариант 1: Vercel Cron (рекомендуется)
1. Создайте файл `vercel.json`:
```json
{
  "crons": [
    {
      "path": "/api/cron/process-notifications",
      "schedule": "*/5 * * * *"
    }
  ]
}
```

#### Вариант 2: Внешний cron сервис
1. Используйте [cron-job.org](https://cron-job.org)
2. Настройте запрос каждые 5 минут:
```
GET https://yourdomain.com/api/cron/process-notifications
Authorization: Bearer YOUR_CRON_SECRET
```

#### Переменные окружения для cron:
```bash
CRON_SECRET=your_secure_random_string
```

## 📊 МОНИТОРИНГ И ЛОГИ

### 1. Логи уведомлений
- Email уведомления логируются в `callback_notifications` таблице
- Статусы: `pending`, `sent`, `failed`
- Ошибки сохраняются в `error_message`

### 2. Аудит действий
- Все действия пользователей логируются в `user_contact_audit`
- Создание пользователей, связывание Telegram, изменения

### 3. Метрики SLA
- Время первого ответа
- Время решения заявки
- Количество попыток связи

## 🔒 БЕЗОПАСНОСТЬ

### 1. RLS (Row Level Security)
- Пользователи видят только свои заявки
- Админы видят все заявки
- API защищены аутентификацией

### 2. Валидация данных
- Проверка email формата
- Проверка телефона
- Санитизация сообщений

### 3. Rate Limiting
- Ограничение на создание заявок
- Защита от спама
