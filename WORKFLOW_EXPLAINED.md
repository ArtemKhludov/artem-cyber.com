# 🚀 Как теперь работает ваш рабочий процесс

## 📋 Полный цикл разработки и деплоя

### 1️⃣ **Работа на локальном сервере**

```bash
# Запускаете локальный сервер
npm run dev

# Сайт доступен на:
http://localhost:3000 или http://localhost:3001
```

**Что происходит:**
- ✅ Работаете в Cursor
- ✅ Изменения видны сразу (Hot Reload)
- ✅ База данных Supabase - та же что и на продакшене
- ✅ Тестируете функционал локально

---

### 2️⃣ **Сохранение изменений в Git (БЕЗ деплоя)**

```bash
# Сохраняете изменения локально
git add .
git commit -m "описание изменений"

# НЕ пушите пока не готовы к деплою!
```

**Что происходит:**
- ✅ Изменения сохранены ЛОКАЛЬНО
- ❌ НЕ загружены на GitHub
- ❌ НЕ задеплоены на сайт
- ✅ Можете продолжать работу

---

### 3️⃣ **Загрузка на сайт (АВТОМАТИЧЕСКИЙ ДЕПЛОЙ)**

```bash
# Когда готовы обновить сайт:
git push origin main
```

**Что происходит автоматически:**

#### Шаг 1: GitHub (5 секунд)
```
git push origin main
   ↓
GitHub получает коммит
   ↓
GitHub отправляет webhook в Vercel
```

#### Шаг 2: GitHub Actions (30-60 секунд)
```
GitHub Actions запускается
   ↓
Проверяет TypeScript (npx tsc --noEmit)
   ↓
Если ошибок НЕТ ✅ - продолжает
Если есть ошибки ❌ - останавливается
```

#### Шаг 3: Vercel (2-3 минуты)
```
Vercel получает webhook от GitHub
   ↓
Клонирует последний код из GitHub
   ↓
Устанавливает зависимости (npm install)
   ↓
Собирает проект (npm run build)
   ↓
Деплоит на energylogic-ai.com
   ↓
Сайт обновлен! 🎉
```

#### Шаг 4: Уведомления
```
Vercel отправляет статус в GitHub
   ↓
Вы видите зеленую галочку ✅ или красный крестик ❌
   ↓
(Опционально) Telegram уведомление
```

**Общее время: 3-4 минуты от push до обновления сайта**

## 🔄 **Разные типы изменений**

### **Изменения в коде (components, pages, API):**
```bash
# 1. Вносите изменения в Cursor
# 2. Тестируете локально (localhost:3000)
# 3. Сохраняете:
git add .
git commit -m "feat: добавил новую функцию"
git push origin main

# 4. Ждете 3-4 минуты
# 5. Сайт обновлен!
```

### **Новые PDF файлы:**
```bash
# Загружаете через админку на сайте:
https://energylogic-ai.com/admin

# ИЛИ загружаете в Supabase Storage напрямую
# НЕ требует деплоя - обновляется мгновенно!
```

### **Новые курсы/программы:**
```bash
# Добавляете через админку:
https://energylogic-ai.com/admin

# ИЛИ добавляете в Supabase таблицу documents
# НЕ требует деплоя - обновляется мгновенно!
```

### **Новые пользователи/покупки:**
```bash
# Создаются автоматически:
# - Регистрация → запись в Supabase
# - Покупка → запись в Supabase
# - Все обновляется в реальном времени

# НЕ требует деплоя!
```

### **Изменения в переменных окружения:**
```bash
# 1. Заходите в Vercel → Settings → Environment Variables
# 2. Добавляете/изменяете переменные
# 3. Нажимаете "Redeploy" в Deployments
# 4. Сайт пересобирается с новыми переменными
```

## ⚠️ **Когда НУЖЕН деплой:**

✅ Изменения в коде (TypeScript, React, компоненты)  
✅ Изменения в API routes  
✅ Изменения в стилях (CSS, Tailwind)  
✅ Изменения в конфигурации (next.config.ts)  
✅ Изменения в middleware  

## ⚠️ **Когда НЕ НУЖЕН деплой:**

❌ Добавление контента в БД (курсы, PDF, пользователи)  
❌ Изменение данных в Supabase  
❌ Загрузка файлов в Supabase Storage  
❌ Обработка платежей  
❌ Обращения в поддержку

## 🔍 **Как отслеживать деплои:**

### **Vercel Dashboard:**
- https://vercel.com/dashboard
- Deployments - список всех деплоев
- Каждый деплой показывает:
  - Коммит (hash и сообщение)
  - Статус (Building/Ready/Error)
  - Время сборки
  - Автор (ваш email)

### **GitHub:**
- https://github.com/ArtemKhludov/EnergyLogic
- Возле каждого коммита будет:
  - ✅ Зеленая галочка - деплой успешен
  - ❌ Красный крестик - деплой с ошибкой
  - 🟡 Желтый круг - деплой в процессе

### **Email уведомления:**
- Vercel отправляет email когда:
  - Деплой успешен ✅
  - Деплой с ошибкой ❌

## 🚨 **Что делать при ошибках деплоя:**

### **Если деплой упал с ошибкой:**

1. **Зайдите в Vercel → Deployments**
2. **Кликните на деплой с ошибкой**
3. **Build Logs → посмотрите ошибку**
4. **Исправьте код локально**
5. **Запушьте исправление:**
   ```bash
   git add .
   git commit -m "fix: исправил ошибку деплоя"
   git push origin main
   ```

### **Если сайт сломался после деплоя:**

1. **Vercel → Deployments**
2. **Найдите последний рабочий деплой** (зеленый)
3. **Три точки (⋮) → Instant Rollback**
4. **Сайт откатится на рабочую версию**
5. **Исправьте ошибку локально**
6. **Задеплойте заново**
