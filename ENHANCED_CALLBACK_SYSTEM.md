# 🚀 Enhanced Callback System - Полная система обратной связи

## 📋 Обзор

Полностью переработанная система обратной связи с интеграцией пользователей, автоматическим созданием аккаунтов, диалогами и уведомлениями.

## ✨ Новые возможности

### 🔄 **Автоматическое создание пользователей**
- При заявке с email автоматически создается пользователь
- Генерируется временный пароль
- Отправляется приветственное письмо с данными для входа
- Связывается заявка с пользователем

### 💬 **Полноценные диалоги**
- Админы могут отвечать на заявки из админки
- Пользователи видят ответы в личном кабинете
- Внутренние заметки для админов
- История переписки

### 🔔 **Умные уведомления**
- Email уведомления о новых ответах
- Telegram уведомления (если подключен)
- Уведомления о смене статуса
- Отслеживание доставки

### 📊 **Расширенная аналитика**
- Статистика по заявкам
- Метрики SLA
- Отслеживание контактов
- Теги и приоритеты

## 🏗️ Архитектура

### **База данных**

#### Новые таблицы:
- `callback_replies` - Ответы на заявки
- `callback_notifications` - Уведомления
- `callback_templates` - Шаблоны ответов
- `callback_sla_metrics` - Метрики SLA

#### Расширенные поля в `callback_requests`:
- `user_id` - Связь с пользователем
- `issue_id` - Связь с обращением
- `auto_created_user` - Автоматически созданный пользователь
- `user_credentials_sent` - Отправлены ли данные для входа
- `admin_notes` - Заметки админа
- `contact_attempts` - Количество попыток связаться
- `priority` - Приоритет (low/medium/high/urgent)
- `assigned_admin_id` - Назначенный админ
- `tags` - Теги
- `metadata` - Дополнительные данные

### **API Endpoints**

#### Новые API:
- `POST /api/callback/enhanced` - Создание заявки с полной интеграцией
- `GET /api/callback/enhanced` - Получение заявок с фильтрами
- `PATCH /api/callback/enhanced` - Обновление заявки
- `GET /api/callback/[id]/replies` - Получение ответов
- `POST /api/callback/[id]/replies` - Создание ответа
- `GET /api/user/callbacks` - Заявки пользователя
- `POST /api/user/callbacks` - Создание заявки пользователем

### **Компоненты**

#### Новые компоненты:
- `UserCallbacksList` - Список заявок в личном кабинете
- `EnhancedCallbacksDashboard` - Расширенная админка
- `CallbackReplyDialog` - Диалог ответа на заявку

## 🔧 Установка и настройка

### **1. Применение миграции**

```sql
-- Выполните SQL из файла enhance_callback_system.sql
\i enhance_callback_system.sql
```

### **2. Переменные окружения**

```env
# Email уведомления
RESEND_API_KEY=your_resend_api_key
NOTIFY_SENDER_EMAIL=noreply@yourdomain.com

# Enhanced Callback System
CALLBACK_SLA_FIRST_RESPONSE_HOURS=4
CALLBACK_SLA_RESOLUTION_HOURS=24
CALLBACK_AUTO_ASSIGN_ENABLED=true
CALLBACK_REMINDER_HOURS=2
```

### **3. Настройка Telegram**

Убедитесь, что настроены:
- `TELEGRAM_BOT_TOKEN` - Токен основного бота
- `TELEGRAM_CHAT_ID` - ID чата для уведомлений
- `TELEGRAM_THREAD_CALLBACKS` - ID потока для заявок

## 🎯 Использование

### **Для пользователей**

1. **Создание заявки через модальное окно:**
   - Заполняют форму на сайте
   - Автоматически создается пользователь (если есть email)
   - Приходит приветственное письмо

2. **Просмотр заявок в личном кабинете:**
   - Переходят в раздел "Заявки"
   - Видят все свои заявки
   - Могут создавать новые заявки
   - Получают уведомления о новых ответах

### **Для администраторов**

1. **Просмотр заявок:**
   - Переходят в `/admin/callbacks/enhanced`
   - Видят все заявки с фильтрами
   - Статистика и метрики

2. **Ответ на заявки:**
   - Нажимают "Ответить" на заявке
   - Пишут ответ (внутренний или для пользователя)
   - Пользователь получает уведомление

3. **Управление статусами:**
   - Меняют статус заявки
   - Назначают админов
   - Добавляют теги и заметки

## 🔄 Workflow

### **Создание заявки:**

```
1. Пользователь заполняет форму
   ↓
2. Сохраняется в callback_requests
   ↓
3. Триггер создает пользователя (если есть email)
   ↓
4. Триггер создает issue_reports
   ↓
5. Отправляется уведомление в Telegram
   ↓
6. Отправляется приветственное письмо
```

### **Ответ на заявку:**

```
1. Админ отвечает в админке
   ↓
2. Сохраняется в callback_replies
   ↓
3. Обновляется статус заявки
   ↓
4. Отправляется уведомление пользователю
   ↓
5. Пользователь видит ответ в личном кабинете
```

## 📊 Метрики и SLA

### **Автоматические метрики:**
- Время до первого ответа
- Время решения заявки
- Количество попыток связаться
- Статистика по приоритетам

### **SLA настройки:**
- `CALLBACK_SLA_FIRST_RESPONSE_HOURS=4` - Первый ответ за 4 часа
- `CALLBACK_SLA_RESOLUTION_HOURS=24` - Решение за 24 часа
