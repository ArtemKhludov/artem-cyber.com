# Инструкция по настройке системы обращений

## Что было реализовано

### ✅ Основные функции:
1. **Система обращений пользователей** - создание и просмотр обращений
2. **Админ-панель** - управление обращениями, ответы, смена статусов
3. **Telegram интеграция** - уведомления админам и пользователям
4. **Email уведомления** - отправка ответов пользователям
5. **Связывание Telegram** - пользователи могут подключить Telegram для уведомлений
6. **Улучшенная админ-панель** - полная информация о пользователях

### ✅ Новые компоненты:
- `TelegramLink` - компонент для связывания Telegram в Dashboard
- `UserIssuesList` - список обращений пользователя
- API для связывания Telegram (`/api/user/telegram/link`)
- Webhook для Telegram бота (`/api/telegram/user-webhook`)

### 1. База данных

### 2. Переменные окружения

Добавьте в `.env.local`:

```bash
# Telegram бот для пользователей
USER_TELEGRAM_BOT_TOKEN=your_user_telegram_bot_token
USER_TELEGRAM_BOT_USERNAME=your_bot_username
USER_TELEGRAM_WEBHOOK_URL=https://yourdomain.com/api/telegram/user-webhook
USER_TELEGRAM_WEBHOOK_SECRET=your_webhook_secret

# Email провайдер
RESEND_API_KEY=your_resend_api_key
NOTIFY_SENDER_EMAIL=noreply@yourdomain.com

# Включить уведомления об обращениях
NOTIFY_ISSUES_VIA_TELEGRAM=true
ISSUE_TELEGRAM_BOT_TOKEN=your_issue_bot_token
ISSUE_TELEGRAM_CHAT_ID=your_issue_chat_id
```

### 3. Настройка Telegram бота

#### Для пользователей:
1. Создайте бота через @BotFather
2. Получите токен и username
3. Настройте webhook:
```bash
curl -X POST "https://api.telegram.org/bot<USER_BOT_TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://yourdomain.com/api/telegram/user-webhook",
    "secret_token": "your_webhook_secret"
  }'
```

#### Для админов (уже настроено):
- Используйте существующий бот для уведомлений о новых обращениях

### 4. Настройка Email

#### Resend (рекомендуется):
1. Зарегистрируйтесь на [resend.com](https://resend.com)
2. Добавьте домен и настройте DNS записи
3. Получите API ключ

#### Альтернативы:
- SendGrid
- AWS SES
- SMTP сервер

## Как работает система

### Пользователь создает обращение:
1. Заходит в Dashboard → Обращения
2. Нажимает "Сообщить о проблеме"
3. Заполняет форму с описанием проблемы
4. Обращение сохраняется в базе данных

### Админ получает уведомление:
1. Telegram уведомление приходит в админский чат
2. В админ-панели появляется новое обращение
3. Админ видит полную информацию о пользователе (имя, телефон, Telegram)

### Админ отвечает:
1. Открывает обращение в админ-панели
2. Видит историю переписки
3. Пишет ответ и отправляет
4. Статус обращения обновляется

### Пользователь получает ответ:
1. **Email уведомление** - приходит на указанный email
2. **Telegram уведомление** - если пользователь подключил Telegram
3. В Dashboard → Обращения видит новый ответ

### Связывание Telegram:
1. Пользователь заходит в Dashboard → Обращения
2. Видит карточку "Telegram уведомления"
3. Нажимает "Подключить Telegram"
4. Открывается Telegram с ссылкой на бота
5. Нажимает "Start" в боте
6. Аккаунт связывается автоматически
