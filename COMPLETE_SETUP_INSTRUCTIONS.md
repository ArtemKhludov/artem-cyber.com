# Полная инструкция по настройке системы ролей и безопасности

## 🎯 Цель
Исправить систему доступа так, чтобы:
1. **Администраторы** имели доступ к админ панели по роли, а не по покупкам
2. **Покупатели** получали доступ только к купленным курсам
3. **Файлы** были правильно разделены на публичные и приватные

## 📋 Шаги выполнения

### Шаг 1: Выполните SQL скрипт в Supabase Dashboard

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Выберите ваш проект
3. Перейдите в раздел **"SQL Editor"**
4. Скопируйте содержимое файла `create-user-roles-system.sql`
5. Вставьте в редактор и нажмите **"Run"**

### Шаг 2: Выполните RLS политики для приватного бакета

1. В том же SQL Editor выполните содержимое файла `setup-private-bucket-rls.sql`
2. Это настроит безопасность для приватного бакета `course-materials-private`

### Шаг 3: Проверьте систему ролей

```bash
node verify-roles-system.js
```

### Шаг 4: Протестируйте флоу покупателя

```bash
node test-customer-flow.js
```

### Шаг 5: Протестируйте изоляцию пользователей

```bash
node test-course-isolation.js
```

## 🔍 Что должно получиться

### Администраторы:
- ✅ Доступ к админ панели без покупки курсов
- ✅ Роль `admin` в таблице `user_profiles`
- ✅ Нет записей в таблице `purchases`

### Покупатели:
- ✅ Роль `customer` в таблице `user_profiles`
- ✅ Доступ только к купленным курсам
- ✅ Приватные файлы доступны только через API
- ✅ Обложки доступны публично

### Безопасность файлов:
- ✅ `course-materials` (публичный) - обложки и превью
- ✅ `course-materials-private` (приватный) - PDF, аудио, видео
- ✅ RLS политики защищают приватные файлы
- ✅ API проверяет покупки перед доступом

## 🚨 Если что-то не работает

1. Проверьте логи в Supabase Dashboard
2. Убедитесь, что все SQL скрипты выполнены успешно
3. Проверьте, что таблицы созданы в разделе "Table Editor"
4. Проверьте RLS политики в разделе "Authentication" → "Policies"
