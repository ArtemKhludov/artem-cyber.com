# ⏰ Настройка Vercel Cron для обработки уведомлений

## 🎯 Цель
Настроить автоматический запуск cron job для обработки email уведомлений каждые 5 минут.

## 📋 Пошаговая инструкция

### Шаг 1: Проверка конфигурации
Файл `vercel.json` уже настроен с расписанием:
```json
{
  "crons": [
    {
      "path": "/api/cron/process-notifications",
      "schedule": "*/5 * * * *"
    }
  ]
}
```

### Шаг 2: Настройка переменных окружения
Убедитесь, что в Vercel Environment Variables установлены:

```env
CRON_SECRET=5788b932b46443d1a00051c923ec61ee231bb60dc7592670df82d16b61159d86
RESEND_API_KEY=your_resend_api_key
NOTIFY_SENDER_EMAIL=noreply@energylogic-ai.com
```

### Шаг 3: Деплой в Vercel
1. Зафиксируйте изменения в Git:
   ```bash
   git add vercel.json
   git commit -m "Add Vercel Cron configuration"
   git push
   ```

2. Vercel автоматически задеплоит изменения

### Шаг 4: Проверка работы Cron
После деплоя проверьте:

1. **В Vercel Dashboard:**
   - Перейдите в раздел "Functions"
   - Найдите функцию "process-notifications"
   - Проверьте логи выполнения

2. **Ручной тест:**
   ```bash
   curl -X GET "https://www.energylogic-ai.com/api/cron/process-notifications" \
     -H "Authorization: Bearer your_cron_secret"
   ```

## ⚙️ Настройки расписания

### Текущее расписание: `*/5 * * * *`
- **Частота:** Каждые 5 минут
- **Время работы:** 24/7
- **Обработка:** До 20 уведомлений за раз

### Альтернативные расписания:

#### Каждую минуту (для тестирования):
```json
"schedule": "* * * * *"
```

#### Каждые 10 минут:
```json
"schedule": "*/10 * * * *"
```

#### Каждые 15 минут:
```json
"schedule": "*/15 * * * *"
```

#### Только в рабочее время (9:00-18:00):
```json
"schedule": "*/5 9-18 * * *"
```

## 🚨 Устранение проблем

### Проблема: Cron не запускается
**Решение:**
1. Проверьте синтаксис в `vercel.json`
2. Убедитесь, что файл задеплоен
3. Проверьте логи в Vercel Dashboard

### Проблема: Ошибка авторизации
**Решение:**
1. Проверьте переменную `CRON_SECRET`
2. Убедитесь, что она совпадает в коде и Vercel
3. Проверьте заголовок Authorization

### Проблема: Уведомления не отправляются
**Решение:**
1. Проверьте `RESEND_API_KEY`
2. Убедитесь, что домен верифицирован
3. Проверьте логи отправки email

### Проблема: Превышение лимитов
**Решение:**
1. Увеличьте интервал между запусками
2. Уменьшите количество обрабатываемых уведомлений
3. Оптимизируйте код обработки
