# 🏠 Руководство по личному кабинету покупателей

## 🎯 Обзор системы

Личный кабинет теперь правильно отображает **только купленные курсы** и предоставляет доступ ко всем материалам каждого курса.

## 🔧 Исправленные проблемы

### ❌ Было:
- Показывались все курсы, а не только купленные
- Неправильная структура данных
- Отсутствие связи между покупками и доступом к файлам

### ✅ Стало:
- Показываются **только купленные курсы**
- Правильная связь между покупками и документами
- Полный доступ ко всем материалам курса

## 📊 Структура данных

### Таблица `purchases`:
- Связывает пользователя с конкретным документом
- Хранит информацию о платеже
- Статус: `completed` = доступ к курсу

### Таблица `orders`:
- Для сессий энергетической диагностики
- Отдельная система от курсов

## 🎨 Новый интерфейс личного кабинета

### 📈 Статистика:
- **Всего покупок**: курсы + сессии
- **Курсы**: только купленные курсы
- **Завершено**: завершенные сессии
- **Потрачено**: общая сумма

### 📚 Вкладка "Мои покупки":
Отображает все покупки (курсы + сессии):

#### Для курсов:
- **Название курса**
- **Тип**: Мини-курс / PDF-документ
- **Цена и статус**
- **Кнопки**: "Открыть курс" / "Просмотр"

#### Для сессий:
- **"Энергетическая диагностика"**
- **Дата и время сессии**
- **Кнопки**: "Скачать отчет" / "Просмотр"

### 🎓 Вкладка "Курсы":
Отображает только купленные курсы с детальной информацией:

#### Информация о курсе:
- **Название и описание**
- **Тип курса** (бейдж)
- **Длительность** (если указана)

#### Доступные материалы:
- **📄 Основной PDF** (всегда есть)
- **📚 X тетрадей** (если есть)
- **🎥 X видео** (если есть)
- **🔊 Аудио** (если есть)

#### Кнопки действий:
- **"Открыть курс"** → `/download/[purchase_id]`
- **"Просмотр"** → `/pdf/[document_id]`

## 🔐 Система доступа

### Принцип работы:
1. **Покупка курса** → запись в `purchases` со статусом `completed`
2. **Авторизация** → проверка email в `purchases`
3. **Доступ к файлам** → только для купленных курсов

### Безопасность:
- Пользователь видит **только свои покупки**
- Доступ к файлам только через авторизованные ссылки
- Проверка прав доступа на сервере

## 🎯 Типы контента

### 1. Мини-курсы (`mini_course`):
- Основной PDF
- Рабочие тетради (множественные)
- Видео-уроки (множественные)
- Аудио-настройка

### 2. PDF-документы (`pdf`):
- Только основной PDF

### 3. Сессии диагностики (`session`):
- PDF-отчет
- Информация о сессии

## 🚀 Преимущества новой системы

### Для покупателей:
- **Четкое понимание** что куплено
- **Полный доступ** ко всем материалам
- **Удобная навигация** по курсам
- **Понятные кнопки** действий

### Для администраторов:
- **Правильная аналитика** покупок
- **Контроль доступа** к контенту

## 📱 Адаптивность

- **Мобильные устройства**: все функции доступны
- **Планшеты**: оптимальное отображение
- **Десктоп**: полный функционал

## 🔧 Технические детали

### API `/api/user/dashboard`:
```typescript
// Возвращает:
{
  user: User,
  purchases: Purchase[], // Все покупки
  courses: Course[], // Только курсы
  orders: Order[], // Только сессии
  stats: {
    totalPurchases: number,
    totalCourses: number,
    completedCourses: number,
    totalSpent: number
  }
}
```

### Структура Purchase:
```typescript
interface Purchase {
  id: string
  product_name: string
  product_type: 'mini_course' | 'pdf' | 'session'
  price: number
  status: 'completed' | 'pending'
  document?: Document // Для курсов
  pdf_url?: string // Для сессий
}
```
