# 🔧 Исправление ошибки API

## ❌ Проблема
```
Ошибка: Document ID, title and file URL are required
```

## ✅ Решение

### 1. Обновлена валидация API
**Файл**: `app/api/admin/workbooks/route.ts`

**Было**:
```typescript
if (!document_id || !title || !file_url) {
  return NextResponse.json(
    { error: 'Document ID, title and file URL are required' },
    { status: 400 }
  )
}
```

**Стало**:
```typescript
if (!document_id || !title) {
  return NextResponse.json(
    { error: 'Document ID and title are required' },
    { status: 400 }
  )
}

// Если file_url не указан, используем заглушку
const finalFileUrl = file_url || 'https://placeholder.com/workbook.pdf'
```

### 2. Обновлены функции быстрого добавления
**Файл**: `components/admin/CoursesManagement.tsx`

**Было**:
```typescript
file_url: '', // Пустой URL, нужно будет загрузить файл
```

**Стало**:
```typescript
file_url: 'https://placeholder.com/workbook.pdf', // Временный URL-заглушка
```

## 🎯 Что изменилось

### API теперь:
- ✅ **Принимает** рабочие тетради без file_url
- ✅ **Автоматически** устанавливает URL-заглушку
- ✅ **Сохраняет** рабочую тетрадь в базе данных
- ✅ **Позволяет** загрузить файл позже

### Быстрые действия теперь:
- ✅ **Работают** без ошибок
- ✅ **Создают** тетради с заглушками
- ✅ **Напоминают** загрузить файл
- ✅ **Обновляют** счетчики

1. **Протестируйте** быстрые действия ➕ и ➖

- **URL-заглушки** используются только для создания тетрадей
- **Реальные файлы** нужно загружать через интерфейс управления
- **Заглушки** не влияют на отображение на сайте
- **Система** работает корректно с временными URL
