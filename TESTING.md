# 🧪 Система тестирования EnergyLogic

Комплексная система тестирования для проверки всех компонентов проекта EnergyLogic.

## 📋 Обзор

Система тестирования включает в себя:

- **Системные тесты** - проверка API, интеграций, базовой функциональности
- **Тесты базы данных** - проверка триггеров, функций, ограничений
- **UI тесты** - проверка доступности страниц и компонентов
- **Тесты потока обратной связи** - полный цикл работы с заявками

## 🚀 Быстрый старт

### Запуск всех тестов
```bash
npm test
```

### Запуск отдельных категорий тестов
```bash
# Системные тесты
npm run test:system

# Тесты базы данных
npm run test:database

# UI тесты
npm run test:ui

# Тесты потока обратной связи
npm run test:callback
```

## ⚙️ Конфигурация

### Переменные окружения

Создайте файл `.env.test` с необходимыми переменными:

```env
# Базовый URL для тестирования
TEST_BASE_URL=http://localhost:3000

# Данные для тестирования
TEST_ADMIN_EMAIL=admin@energylogic-ai.com
TEST_USER_EMAIL=test@example.com
TEST_USER_PHONE=+1234567890

# Telegram (опционально)
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id

# Supabase
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_key
```

### Запуск с переменными окружения
```bash
# Загрузка переменных из .env.test
source .env.test && npm test

# Или с конкретными переменными
TEST_BASE_URL=https://your-domain.com npm test
```

## 🔧 Настройка для CI/CD

### GitHub Actions

Создайте файл `.github/workflows/test.yml`:

```yaml
name: Tests

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
    
    - name: Run tests
      run: npm test
      env:
        TEST_BASE_URL: ${{ secrets.TEST_BASE_URL }}
        TEST_USER_EMAIL: ${{ secrets.TEST_USER_EMAIL }}
        TELEGRAM_BOT_TOKEN: ${{ secrets.TELEGRAM_BOT_TOKEN }}
        TELEGRAM_CHAT_ID: ${{ secrets.TELEGRAM_CHAT_ID }}
        NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.NEXT_PUBLIC_SUPABASE_URL }}
        NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.NEXT_PUBLIC_SUPABASE_ANON_KEY }}
        SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.SUPABASE_SERVICE_ROLE_KEY }}
    
    - name: Upload test results
      uses: actions/upload-artifact@v3
      if: always()
      with:
        name: test-results
        path: |
          test-*.json
          test-results.json
```

### Vercel

Для автоматического тестирования на Vercel добавьте в `vercel.json`:

```json
{
  "buildCommand": "npm run build && npm test",
  "installCommand": "npm ci"
}
```

## 🐛 Отладка

### Логирование

Все тесты выводят подробные логи:

- `ℹ` - информационные сообщения
- `✓` - успешные тесты
- `✗` - проваленные тесты
- `⚠` - предупреждения
- `🧪` - запуск тестов
- `📊` - результаты

### Частые проблемы

1. **Timeout ошибки** - увеличьте таймауты в тестах
2. **Ошибки подключения** - проверьте URL и доступность сервисов
3. **Ошибки аутентификации** - проверьте токены и ключи
4. **Ошибки базы данных** - проверьте миграции и триггеры

### Отладка отдельных тестов

```bash
# Запуск с подробными логами
DEBUG=* npm run test:system

# Запуск конкретного теста
node scripts/test-system.js
```

## 📈 Мониторинг

### Метрики

Система тестирования отслеживает:

- Время выполнения тестов
- Процент успешности
- Количество проваленных тестов
- Детали ошибок

### Алерты

Настройте алерты для:

- Провала более 10% тестов
- Критических ошибок (база данных, API)
- Долгого времени выполнения тестов

## 🔄 Обновление тестов

### Добавление новых тестов

1. Создайте новый тест в соответствующем файле
2. Добавьте его в метод `runAllTests()`
3. Обновите документацию
4. Запустите тесты для проверки

### Модификация существующих тестов

1. Внесите изменения в тест
2. Проверьте, что тест все еще проходит
3. Обновите документацию при необходимости
