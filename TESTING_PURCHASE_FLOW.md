# 🧪 Тестирование системы покупок и доступа

## 🎯 Цель тестирования

Убедиться, что:
1. **Покупатель получает доступ только к купленному курсу**
2. **Личный кабинет отображает правильную информацию**
3. **Все файлы курса доступны после покупки**

## 📋 Чек-лист тестирования

### 1. 🔐 Авторизация
- [ ] Пользователь может войти в систему
- [ ] Сессия сохраняется между перезагрузками
- [ ] Неавторизованный пользователь перенаправляется на логин

### 2. 🛒 Процесс покупки
- [ ] Пользователь может выбрать курс
- [ ] Корректно создается запись в `purchases`
- [ ] Статус покупки `pending` → `completed` после оплаты
- [ ] Email пользователя сохраняется в `user_email`

### 3. 🏠 Личный кабинет
- [ ] Отображаются только купленные курсы
- [ ] Показывается правильная статистика
- [ ] Курсы группируются по типам
- [ ] Отображаются все доступные материалы

### 4. 📚 Доступ к материалам
- [ ] Кнопка "Открыть курс" ведет на `/download/[purchase_id]`
- [ ] Кнопка "Просмотр" ведет на `/pdf/[document_id]`
- [ ] Все файлы курса доступны (PDF, тетради, видео, аудио)
- [ ] Нет доступа к некупленным курсам

## 🧪 Тестовые сценарии

### Сценарий 1: Новый пользователь
1. **Регистрация** нового пользователя
2. **Покупка** одного курса
3. **Проверка** личного кабинета
4. **Ожидаемый результат**: 1 курс в дашборде

### Сценарий 2: Пользователь с несколькими покупками
1. **Покупка** 2-3 разных курсов
2. **Проверка** отображения всех курсов
3. **Проверка** статистики
4. **Ожидаемый результат**: все курсы видны, статистика корректна

### Сценарий 3: Мини-курс с полным контентом
1. **Покупка** мини-курса с тетрадями, видео, аудио
2. **Проверка** отображения всех материалов
3. **Проверка** доступа к каждому файлу
4. **Ожидаемый результат**: все материалы доступны

### Сценарий 4: Безопасность доступа
1. **Попытка** доступа к некупленному курсу
2. **Проверка** блокировки доступа
3. **Ожидаемый результат**: доступ запрещен

## 🔍 Что проверить в админ-панели

### Таблица `purchases`:
- [ ] Записи создаются при покупке
- [ ] `user_email` заполняется корректно
- [ ] `document_id` связывается с правильным курсом
- [ ] `payment_status` обновляется после оплаты

### Таблица `documents`:
- [ ] Все курсы имеют правильный `course_type`
- [ ] Флаги `has_workbook`, `has_videos`, `has_audio` корректны
- [ ] Счетчики `video_count`, `workbook_count` актуальны

## 🐛 Возможные проблемы

### Проблема 1: Курсы не отображаются
**Причина**: Неправильная связь в API
**Решение**: Проверить запрос в `/api/user/dashboard`

### Проблема 2: Доступ к файлам заблокирован
**Причина**: Неправильные права доступа в Supabase
**Решение**: Проверить RLS политики

### Проблема 3: Статистика неверная
**Причина**: Ошибка в подсчете данных
**Решение**: Проверить логику в API

## 📊 Тестовые данные

### Создать тестовые покупки:
```sql
-- Тестовая покупка мини-курса
INSERT INTO purchases (
  document_id,
  user_email,
  payment_status,
  amount_paid,
  currency
) VALUES (
  'document_id_here',
  'test@example.com',
  'completed',
  5000,
  'RUB'
);
```

### Проверить данные:
```sql
-- Проверить покупки пользователя
SELECT p.*, d.title, d.course_type 
FROM purchases p
JOIN documents d ON p.document_id = d.id
WHERE p.user_email = 'test@example.com';
```
