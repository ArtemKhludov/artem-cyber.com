# Руководство по тестированию системы обращений

## Что нужно протестировать

### 1. Создание обращения пользователем

#### Шаги:
1. Войдите как пользователь в Dashboard
2. Перейдите в "Мои покупки"
3. Нажмите "Сообщить о проблеме" на любой покупке
4. Заполните форму:
   - Тема: "Тестовая проблема"
   - Описание: минимум 30 символов
   - Email: ваш email
5. Нажмите "Отправить"

#### Ожидаемый результат:
- ✅ Обращение создается
- ✅ Админ получает Telegram уведомление с полной информацией о пользователе
- ✅ В админ-панели появляется новое обращение

### 2. Telegram уведомления для админов

#### Проверьте:
- ✅ Приходит сообщение в Telegram
- ✅ В сообщении указано имя пользователя (если есть)
- ✅ Указан телефон (если есть)
- ✅ Указан email
- ✅ Указана информация о покупке/документе
- ✅ Есть ссылка на проблему

### 3. Админ-панель

#### Проверьте:
1. Перейдите в админ-панель
2. Найдите созданное обращение
3. Откройте его для ответа

#### Ожидаемый результат:
- ✅ В таблице отображается имя пользователя
- ✅ Показаны контакты (телефон, Telegram если есть)
- ✅ В диалоге ответа есть карточка с полной информацией о пользователе
- ✅ Можно написать ответ и отправить

### 4. Ответ админа

#### Шаги:
1. В админ-панели откройте обращение
2. Напишите ответ (минимум 3 символа)
3. Нажмите "Отправить"

#### Ожидаемый результат:
- ✅ Ответ сохраняется в базе данных
- ✅ Пользователь получает email уведомление
- ✅ Пользователь получает Telegram уведомление (если подключен)
- ✅ Статус обращения обновляется

### 5. Связывание Telegram

#### Шаги:
1. Войдите как пользователь в Dashboard
2. Перейдите в "Обращения"
3. Найдите карточку "Telegram уведомления"
4. Нажмите "Подключить Telegram"
5. Откроется Telegram с ссылкой на бота
6. Нажмите "Start" в боте

#### Ожидаемый результат:
- ✅ Аккаунт успешно связывается
- ✅ Пользователь получает подтверждение в Telegram
- ✅ В Dashboard показывается статус "Telegram аккаунт связан"

### 6. Просмотр обращений пользователем

#### Шаги:
1. В Dashboard → Обращения
2. Найдите раздел "Мои обращения"
3. Откройте созданное обращение

#### Ожидаемый результат:
- ✅ Отображается история переписки
- ✅ Видны ответы админа
- ✅ Показывается статус обращения

## Возможные проблемы и решения

### Проблема: Telegram уведомления не приходят

#### Проверьте:
1. Переменные окружения:
   ```bash
   TELEGRAM_BOT_TOKEN=ваш_токен
   TELEGRAM_CHAT_ID=ваш_chat_id
   ```

2. Логи в консоли браузера и сервера

3. Доступность Telegram API

### Проблема: В админке не отображаются данные пользователей

#### Проверьте:
1. Выполнена ли миграция `add_telegram_fields.sql`
2. Есть ли поля в таблице `users`:
   - `name`
   - `phone` 
   - `telegram_username`

### Проблема: Email уведомления не работают

#### Проверьте:
1. Переменные окружения:
   ```bash
   RESEND_API_KEY=ваш_api_key
   NOTIFY_SENDER_EMAIL=ваш_email
   ```

2. Настройки домена в Resend

### Проблема: Связывание Telegram не работает

#### Проверьте:
1. Переменные окружения:
   ```bash
   USER_TELEGRAM_BOT_TOKEN=ваш_токен
   USER_TELEGRAM_BOT_USERNAME=имя_бота
   ```

2. Настроен ли webhook для бота

## Переменные окружения для тестирования

```bash

TELEGRAM_CHAT_ID=ваш_chat_id
TELEGRAM_THREAD_ISSUES=thread_id_для_обращений

# Бот для пользователей
USER_TELEGRAM_BOT_TOKEN=8289281549:AAHnCv4FRz62TBJN-wjPdt_kZ6T3tBj-sgY
USER_TELEGRAM_BOT_USERNAME=имя_вашего_бота
USER_TELEGRAM_WEBHOOK_URL=https://yourdomain.com/api/telegram/user-webhook
USER_TELEGRAM_WEBHOOK_SECRET=ваш_секретный_токен

# Email
RESEND_API_KEY=ваш_resend_api_key
NOTIFY_SENDER_EMAIL=noreply@yourdomain.com
```
